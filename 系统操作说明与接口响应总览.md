# 集装箱码头管理系统操作说明、接口说明与服务器端响应总览

## 1. 文档说明

本文档用于说明“集装箱码头管理系统”的整体使用流程、模块操作方法、后端接口设计、请求参数和服务器端响应格式。

系统后端采用 Flask 提供 HTTP 接口，前端页面通过 `fetch` 与服务器通信，核心数据存储在 SQLite 数据库中。

项目主要目录：

```text
Container-terminal-management-system/
├─ Management/Container/app.py                 # Flask 应用入口、登录认证、权限控制
├─ Management/Container/models/container_model.py
├─ Management/Container/routes/                # 各业务模块接口
├─ pages/                                      # 前端页面
├─ js/                                         # 前端交互脚本
├─ css/                                        # 样式文件
└─ detect/                                     # YOLOv5 + LPRNet 车牌识别工程
```

默认服务地址：

```text
http://127.0.0.1:5000
```

系统页面：

| 页面 | 文件 | 功能 |
|---|---|---|
| 登录页 | `login.html` | 用户登录 |
| 首页/驾驶舱 | `index.html` | 系统总览、码头地图、KPI |
| 集装箱管理 | `pages/container-management.html` | 箱信息维护、状态流转 |
| 堆场管理 | `pages/yard-management.html` | 堆场维护、箱位分配、智能分配 |
| 船舶计划 | `pages/ship-plan-management.html` | 船舶维护、清单导入、后台自动作业 |
| 码头作业 | `pages/terminal-operations.html` | 作业单维护、状态推进 |
| 进口闭环 | `pages/import-lifecycle.html` | 放行、预约、闸口、提箱、异常 |
| 设备管理 | `pages/equipment-management.html` | 设备维护、任务分配、AGV 调度 |
| 财务计费 | `pages/finance-billing.html` | 账单生成、结算 |
| 危险品管理 | `pages/dangerous-management.html` | 危险品箱合规检查与纠偏 |

## 3. 通用接口规范

### 3.1 JSON 请求

大部分接口使用 JSON：

```http
Content-Type: application/json
```

前端通用请求方式：

```js
fetch('/containers', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify(payload)
})
```

`credentials: 'include'` 或 `credentials: 'same-origin'` 用于携带登录 Cookie。

### 3.2 文件上传请求

船舶清单导入和车牌视觉识别使用 `multipart/form-data`：

```js
const formData = new FormData();
formData.append('file', excelFile);

fetch('/ships/import_manifest', {
  method: 'POST',
  credentials: 'include',
  body: formData
});
```

上传文件时不要手动设置 `Content-Type`，浏览器会自动生成 boundary。

### 3.3 通用成功响应

创建成功：

```json
{
  "message": "新增成功",
  "data": {}
}
```

查询成功：

```json
[
  {}
]
```

操作成功：

```json
{
  "message": "操作成功",
  "data": {}
}
```

### 3.4 通用失败响应

参数错误：

```json
{
  "message": "缺少必填字段"
}
```

唯一约束冲突：

```json
{
  "message": "箱号已存在"
}
```

资源不存在：

Flask `get_or_404` 会返回 `404` 页面或 404 响应。

常见状态码：

| 状态码 | 含义 |
|---|---|
| `200` | 请求成功 |
| `201` | 创建成功 |
| `400` | 参数错误或业务规则校验失败 |
| `401` | 未登录 |
| `403` | 无权限 |
| `404` | 资源不存在 |
| `409` | 唯一约束冲突或数据保存冲突 |
| `500` | 服务器内部错误 |

## 5. 集装箱管理模块

### 5.1 操作说明

进入：

```text
首页 -> 集装箱管理
```

主要操作：

1. 查看全部集装箱。
2. 新增集装箱。
3. 编辑箱号、箱型、重空、危险品、冷藏、堆场位置等信息。
4. 删除集装箱。
5. 推进集装箱状态。

状态流转规则：

```text
在船上/在船中 -> 已卸船 -> 转运中 -> 堆场存储 -> 等待提箱 -> 已装车待出闸 -> 离港
在堆 -> 待装船 -> 离港
```

危险品规则：

- 危险品集装箱必须存放在危险品堆场。
- 非危险品集装箱不能存放在危险品堆场。

## 14. 全系统典型操作链路

### 14.1 进口船舶卸船入堆链路

```text
1. 登录系统
2. 船舶计划 -> 新增船舶
3. 船舶计划 -> 导入 Excel 任务清单
4. 堆场管理 -> 按船舶智能分配箱位
5. 船舶计划 -> 启动后台自动作业
6. 系统模拟岸桥卸船、AGV转运、场桥入堆
7. 首页/码头作业查看任务状态
8. 设备管理查看设备状态
```

### 14.2 手动作业调度链路

```text
1. 码头作业 -> 新增作业单
2. 设备管理 -> 选择匹配类型设备
3. 分配任务
4. 任务状态变为 in-progress
5. 释放设备或推进任务状态
6. 任务完成后集装箱状态同步更新
```

### 14.3 进口提箱链路

```text
1. 进口闭环 -> 海关放行
2. 创建预约，绑定车牌和箱号
3. 闸口视觉识别进闸或人工进闸
4. 堆场提箱
5. 闸口视觉识别出闸或人工出闸
6. 集装箱离港
7. 财务计费 -> 生成账单并结算
```

### 14.4 危险品合规链路

```text
1. 堆场管理 -> 创建危险品堆场
2. 集装箱管理 -> 标记危险品箱
3. 危险品管理 -> 查看违规箱
4. 一键重新分配
5. 系统将危险品箱调整到危险品堆场
```

## 16. 常见问题与服务器响应

### 16.1 未登录

```json
{
  "message": "请先登录"
}
```

处理：

```text
返回登录页，重新登录。
```

### 16.2 无权限

```json
{
  "message": "当前角色无权执行该操作"
}
```

处理：

```text
切换管理员或具备对应权限的角色。
```

### 16.3 数据重复

```json
{
  "message": "箱号已存在"
}
```

```json
{
  "message": "设备编号已存在"
}
```

```json
{
  "message": "账单号已存在"
}
```

处理：

```text
修改唯一字段，如箱号、设备编号、账单号。
```

### 16.4 危险品堆场违规

```json
{
  "message": "危险品集装箱必须存放在危险品堆场"
}
```

处理：

```text
先创建危险品堆场，再分配危险品箱。
```

### 16.5 目标箱位占用

```json
{
  "message": "目标箱位已被占用",
  "container": {}
}
```

处理：

```text
选择其他区域、列、层，或使用智能分配。
```

### 16.6 作业顺序不合法

```json
{
  "message": "当前作业阶段不允许直接完成"
}
```

处理：

```text
按岸桥卸船、AGV转运、场桥入堆等顺序推进作业。
```

### 16.7 车牌识别失败

```json
{
  "message": "车牌视觉识别失败：YOLOv5 + LPRNet 车牌识别失败：YOLOv5 未检测到车牌区域"
}
```

处理：

```text
检查图片清晰度、模型依赖、权重文件；必要时使用人工进闸/出闸。
```

## 17. 建议演示顺序

用于课程设计答辩时，可以按以下顺序演示：

```text
1. 登录系统，展示不同角色菜单权限。
2. 首页查看码头 KPI 和堆场/设备状态。
3. 新增船舶，导入船舶清单。
4. 启动后台自动作业，展示岸桥、AGV、场桥流程。
5. 查看作业单状态和设备状态。
6. 查看堆场箱位，演示智能分配。
7. 进入进口闭环，完成放行、预约、视觉进闸、提箱、视觉出闸。
8. 演示视觉失败时人工备用进出闸。
9. 演示危险品违规检查和重新分配。
10. 进入财务模块生成账单并结算。
```
